# Indic ANS Compressor

A Rust-based text compression library targeting Indic scripts (Hindi, Tamil, Telugu, Bengali, and more), compiled to WebAssembly for use in the browser.

## Features

- **Lossless compression** — perfect round-trip reconstruction using a vocabulary-sorted token stream
- **Lossy compression** — phonetically-aware normalization at four quality levels (Very High / High / Medium / Low)
- **Neural-hybrid mode** — a lightweight (~50 KB) script-tuned neural predictor blended with corpus frequency scoring for improved token ordering
- **Progressive decoding** — decode only the first N chunks for fast previews
- **Multi-script support** — pre-tuned models for Devanagari, Tamil, Telugu, Kannada, Malayalam, Bengali, Gujarati, Punjabi, Odia, and Sinhala
- **Browser UI** — self-contained web frontend with drag-and-drop file upload, sliders, and download

## How It Works

1. **Tokenise** — the input is split into word + space tokens. Repeated words share a single ID.
2. **Frequency sort** — tokens are ranked by corpus frequency (optionally blended with a neural prior) and assigned compact IDs (most-common = smallest ID).
3. **VarInt encode** — token IDs are written with a 1–5 byte variable-length integer scheme optimized for small values.
4. **Format selection** — if the IC (compressed) stream is smaller than raw UTF-8, it is used; otherwise the encoder falls back to a UC (uncompressed) passthrough.
5. **Lossy pre-processing** (optional) — whitespace normalization, punctuation folding, or comma/semicolon removal run before step 1 and are tagged in the stream header so the decoder can report quality faithfully.

### Stream Format

| Magic | Meaning |
|-------|---------|
| `UC`  | Uncompressed UTF-8 |
| `UL`  | Uncompressed + lossy metadata |
| `IC`  | Compressed token stream |
| `IL`  | Compressed + lossy metadata |

## Getting Started

### Prerequisites

- [Rust](https://www.rust-lang.org/tools/install) (stable, 2021 edition)
- [wasm-pack](https://rustwasm.github.io/wasm-pack/installer/) for building the WebAssembly target
- A local HTTP server to serve the web UI (e.g. `npx serve`, Python's `http.server`, or VS Code Live Server)

### Build

```bash
# Build the WASM package into web/pkg/
wasm-pack build --target web --out-dir web/pkg

# Serve the web UI
cd web
npx serve .
# or
python3 -m http.server 8080
```

Open `http://localhost:8080` in your browser.

### Run as a CLI (optional)

The crate also builds as a native binary. Add a `[[bin]]` section to `Cargo.toml` pointing at a `src/main.rs` entry point, then:

```bash
cargo build --release
./target/release/indic_ans_compressor
```

## API

### WASM (JavaScript)

```js
import init, {
  encode_stream_wasm,
  encode_stream_advanced_wasm,
  decode_full_wasm,
  decode_prefix_wasm,
  estimate_lossy_savings,
  get_compression_info,
} from './pkg/indic_ans_compressor.js';

await init();

// Basic lossless compression
const compressed = encode_stream_wasm(text, chunkSize);

// Advanced: neural + lossy
const compressed = encode_stream_advanced_wasm(
  text,
  chunkSize,    // tokens per chunk, e.g. 40
  useNeural,    // boolean
  neuralWeight, // 0.0–1.0, e.g. 0.3
  qualityLevel  // 0=Lossless, 1=VeryHigh, 2=High, 3=Low
);

const decoded = decode_full_wasm(compressed);
const partial  = decode_prefix_wasm(compressed, nChunks);
```

### Rust

```rust
use indic_ans_compressor::{encode_stream, decode_full, CompressionOptions};

let bytes   = encode_stream("नमस्ते दुनिया", 40)?;
let decoded = decode_full(&bytes)?;
assert_eq!(decoded, "नमस्ते दुनिया");
```

## Compression Quality Levels

| Level | Value | Transformations Applied |
|-------|-------|------------------------|
| Lossless | 0 | None — perfect reconstruction |
| Very High | 1 | Whitespace normalization only |
| High | 2 | Whitespace + `...`→`…`, `!!`→`!`, `??`→`?` |
| Low | 3 | High + strip commas and semicolons |
